## Changes applied (from this report)

The following items from the audit were implemented:

- **Migration drift**: Registered missing migrations in `scripts/run-migrate.mjs`: `initiatives_brand_template_columns`, `email_campaign_schema_ensure`, `artifact_verifications`, `upload_bucket_and_policies`, `introducing_emma_contract`, `fix_introducing_emma_hero_placeholder` (replaced non-existent `brand_google_credentials`), `intent_type_email_design_generator`, `seo_url_risk_snapshots`, `phase6_durable_graph_runtime`. Run-migrate now covers all 52 migration files that are intended for the Control Plane runner.
- **Health endpoints**: Added `GET /health/migrations` (migration status note) and `GET /health/schema` (DB check + pointer to `/v1/schema_drift`).
- **OAuth CSRF**: Documented that state is validated for CSRF in `GET /v1/seo/google/callback` and in `control-plane/src/seo-google-oauth.ts` (`handleOAuthCallback` / `decodeState`).
- **Client error reporting**: Added `POST /v1/errors` stub (accepts JSON body, returns 202 Accepted).
- **Duplicate /health**: Audit script reported duplicate `/health`; only one `GET /health` exists (plus `/health/db`, `/v1/health`). No change made.

**Second pass (implemented):**

- **Rate limiting:** Added `express-rate-limit`: general API 300/min (configurable via `RATE_LIMIT_MAX_PER_MIN`), auth paths (e.g. `/v1/seo/google/*`, `*google_access_token*`, `*google_credentials*`) 30 per 15 min. See `control-plane/src/api.ts`.
- **MCP servers (docs):** Added **docs/MCP_SERVERS_RECOMMENDED.md** with recommended GitHub, Vercel, and Supabase MCPs and Cursor config snippets (env, security notes).
- **Migration naming:** Added **docs/MIGRATION_NAMING.md** (convention `YYYYMMDD00000_snake_case.sql`) and **scripts/validate-migration-names.mjs**; npm script `npm run verify:migration-names`.
- **Schema drift (full):** Implemented **control-plane/src/schema-drift.ts** (fetch current schema from `information_schema`, store snapshot, diff). New migration **20250331000013_schema_snapshots.sql** and registration in run-migrate. **GET /v1/schema_drift** returns current, stored, and diff; **POST /v1/schema_drift/capture** stores a baseline (e.g. after deploy).
- **Schema drift alerts:** Control Plane runs a **10-minute loop** that calls `computeSchemaDrift`; if drift is detected, logs a warning and records to **incident_memory** (signature `schema_drift`). See `control-plane/src/index.ts` → `startSchemaDriftAlertLoop()`.
